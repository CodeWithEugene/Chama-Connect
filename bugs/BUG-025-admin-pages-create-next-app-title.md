# BUG-025 — Admin/dashboard pages also ship with `<title>Create Next App</title>` — logged-in users' browser tabs are unusable

| Field | Value |
|---|---|
| Severity | Medium |
| Surface | Auth / dashboard / UX |
| Status | Open |
| Discovered | 2026-04-20 |
| Discovered by | Playwright recon — DOM grep of authenticated pages |

## Evidence

Every crawled authenticated page ships with the default Next.js boilerplate title:

```
dash_01__chamaconnect_io_admin_dashboard.html              : <title>Create Next App</title>
dash_02__chamaconnect_io_admin_chamas.html                 : <title>Create Next App</title>
dash_03__chamaconnect_io_admin_dashboard_notifications.html: <title>Create Next App</title>
dash_04__chamaconnect_io_admin_chamas_create.html          : <title>Create Next App</title>
landing__chamaconnect_io_admin_dashboard.html              : <title>Create Next App</title>
```

Evidence: `recon/artifacts/2026-04-20T09-40-50-508Z/html/dash_*.html`.

## Scope vs BUG-001

BUG-001 ("Every public page ships with default Next.js boilerplate metadata") is explicitly scoped to **public** routes and names them one by one. This bug covers the mirror problem on **authenticated** routes. A developer fixing BUG-001 by adding per-route metadata exports to `app/(marketing)/*/page.tsx` can very easily stop there and leave `app/(admin)/**/page.tsx` untouched — which is exactly the state of the live site today.

## User impact

A logged-in chama treasurer who has five chama tabs open — say Dashboard, My Chamas, a specific chama's Loans, Notifications, and Create Chama — sees five identical tabs titled **"Create Next App"**. Nothing in the tab label tells them which chama or screen is which. Every navigation is guess-and-click.

1. **Browser tabs are unusable for multi-tab workflows.** Treasurers routinely compare two chamas side-by-side (e.g. reconciling contributions between "Umoja SACCO" and "Westlands Merry-Go-Round"). With identical tab titles they lose the tab.
2. **History / bookmarks are unusable.** Pressing Ctrl-H / Cmd-Y shows `Create Next App — chamaconnect.io` for every visited admin page.
3. **Screen readers announce the page title on navigation.** A blind treasurer using JAWS hears "Create Next App, Create Next App, Create Next App" as they move between sections, with no indication of what changed.
4. **Monitoring / analytics join titles to pageviews.** If Datadog RUM, Mixpanel, or PostHog are installed, every admin pageview lands on the same title in their funnel reports.
5. **Trust signal** — the same logged-in treasurer who is about to approve a KES 200,000 payout will notice the title "Create Next App" in their tab. For a money platform this is not a good visual.

## Root cause

Same as BUG-001: the root `app/layout.tsx` exports a default `metadata: { title: "Create Next App", description: "Generated by create next app" }` that was never customized. Admin pages do not override it, so they inherit the default. The `/about` page has its own per-route metadata export — the only page on the whole site with correct metadata.

## Proposed fix

Ship the title fix in two layers:

1. **Fix the root layout** (this is the same change BUG-001 proposes; it is the lowest-cost single point of fix):

```tsx
// app/layout.tsx
export const metadata: Metadata = {
  metadataBase: new URL("https://chamaconnect.io"),
  title: {
    default: "ChamaConnect — Secure, Transparent Table Banking for Kenya",
    template: "%s · ChamaConnect",
  },
  description:
    "ChamaConnect is a blockchain-powered financial management platform for Kenyan chamas and SACCOs.",
  icons: { icon: "/favicon.ico" },
  robots: { index: true, follow: true, googleBot: { index: true, follow: true } },
};
```

That alone makes every admin page render `ChamaConnect — ...` instead of `Create Next App`.

2. **Add per-route metadata to each admin page** so the *tab* is contextual:

```tsx
// app/admin/dashboard/page.tsx
export const metadata = { title: "Dashboard" };
// app/admin/chamas/page.tsx
export const metadata = { title: "My Chamas" };
// app/admin/dashboard/notifications/page.tsx
export const metadata = { title: "Notifications" };
// app/admin/chamas/create/page.tsx
export const metadata = { title: "Create Chama" };

// for dynamic routes use generateMetadata:
// app/admin/chamas/[id]/page.tsx
export async function generateMetadata({ params }) {
  const chama = await getChama(params.id);
  return { title: chama?.name ? `${chama.name}` : "Chama" };
}
```

3. **Add `noindex` on admin routes** so they never accidentally get SEO-indexed:

```tsx
// app/admin/layout.tsx
export const metadata: Metadata = {
  robots: { index: false, follow: false, googleBot: { index: false, follow: false } },
};
```

4. **Set `robots: index:false`** specifically on `admin/chamas/[id]/*` even when the chama name fix ships, so private chama names never leak into search.

## Verification

- `curl -sSL https://chamaconnect.io/admin/dashboard | grep -oE '<title>[^<]+</title>'` → `<title>Dashboard · ChamaConnect</title>`.
- `curl -sSL https://chamaconnect.io/admin/chamas/create | grep -oE '<title>[^<]+</title>'` → `<title>Create Chama · ChamaConnect</title>`.
- Five admin tabs open in the same browser — the Chrome tab bar shows five distinct titles.
- Playwright regression across admin surfaces:

```ts
test("no admin page ships with 'Create Next App' title", async ({ page }) => {
  for (const p of ["/admin/dashboard", "/admin/chamas", "/admin/chamas/create", "/admin/dashboard/notifications"]) {
    await page.goto(p);
    const t = await page.title();
    expect(t).not.toMatch(/Create Next App/i);
    expect(t).toMatch(/ChamaConnect/i);
  }
});
```
