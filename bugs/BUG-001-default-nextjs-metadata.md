# BUG-001 — Every public page ships with default Next.js boilerplate metadata

| Field | Value |
|---|---|
| Severity | High |
| Surface | Public site / SEO / social sharing |
| Status | Open |
| Discovered | 2026-04-20 |
| Discovered by | Manual (curl) |

## Evidence

- URLs affected (all verified with `curl -sSL https://chamaconnect.io/<path>`):
  - `/` — `<title>Create Next App</title>` · `<meta name="description" content="Generated by create next app"/>`
  - `/features` — same
  - `/pricing` — same
  - `/faqs` — same
  - `/contact` — same
  - `/community` — same
  - `/terms` — same
  - `/onboard-chama` — same
  - `/get-started` (login) — same
  - `/about` — the **only** correctly-titled page (`<title>About — ChamaConnect</title>`), which shows the team knows how to do it but forgot on the rest
- Confirmed by inspecting `/tmp/cc_*.html` saved on 2026-04-20.

```html
<!-- Actual HTML shipped on https://chamaconnect.io/ as of 2026-04-20 -->
<title>Create Next App</title>
<meta name="description" content="Generated by create next app"/>
```

## User impact

This is the single most damaging visible bug on the site.

1. **Google / Bing search results** show the title "Create Next App" with description "Generated by create next app". For a product that needs founder-led SEO to acquire chamas, this is a death sentence — no Kenyan admin searching "chama app M-Pesa" will click that result.
2. **WhatsApp / Facebook / Twitter / LinkedIn link previews** render the default. A growth loop for chamas is admins sharing the app into WhatsApp groups; the preview currently says "Create Next App". That is a massive trust-killer for a financial product.
3. **Browser tab / bookmark label** shows "Create Next App" — a user who bookmarks the site can't find it again.

## Root cause

The page-level `metadata` export was never customised in Next.js 15 App Router. Default is inherited from `app/layout.tsx`, which still has:

```tsx
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
```

The `/about` page is the only one with a per-route metadata export, which is why it is the only correct one.

## Proposed fix

Two-step fix:

1. Replace the root `metadata` export in `app/layout.tsx`:

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  metadataBase: new URL("https://chamaconnect.io"),
  title: {
    default: "ChamaConnect — Secure, Transparent Table Banking for Kenya",
    template: "%s | ChamaConnect",
  },
  description:
    "ChamaConnect is a blockchain-powered financial management platform for Kenyan chamas and SACCOs. Track contributions, manage loans, and generate audit-proof reports in real time.",
  applicationName: "ChamaConnect",
  keywords: [
    "chama",
    "SACCO",
    "table banking",
    "Kenya",
    "M-Pesa",
    "group savings",
    "merry-go-round",
    "blockchain",
  ],
  openGraph: {
    type: "website",
    locale: "en_KE",
    url: "https://chamaconnect.io",
    siteName: "ChamaConnect",
    title: "ChamaConnect — The Future of Chamas",
    description:
      "Secure, transparent, automated financial management for African savings groups.",
    images: [
      {
        url: "/images/og-cover.png",
        width: 1200,
        height: 630,
        alt: "ChamaConnect",
      },
    ],
  },
  twitter: {
    card: "summary_large_image",
    title: "ChamaConnect — The Future of Chamas",
    description:
      "Secure, transparent, automated financial management for African savings groups.",
    images: ["/images/og-cover.png"],
  },
  icons: {
    icon: "/favicon.ico",
    apple: "/apple-touch-icon.png",
  },
  alternates: { canonical: "https://chamaconnect.io" },
};
```

2. Add per-route `metadata` exports so every route has its own title/description. Example for `/features`:

```tsx
// app/features/page.tsx
export const metadata = {
  title: "Features",
  description:
    "Group & membership management, smart financial management, blockchain-backed reporting, automated policies, and M-Pesa integration — all in one chama platform.",
};
```

3. Add a 1200×630 `public/images/og-cover.png` (ChamaConnect logo on brand-green background + tagline) so link previews are branded.
